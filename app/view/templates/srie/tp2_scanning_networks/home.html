SRIE > TP2 - Scanning Networks

Scanning a network is a fundamental practice in cybersecurity that involves actively probing and examining the target network to identify open ports, services, devices, and potential vulnerabilities.

Useful tools used in this phase

Below is a list of tools that can be used for this pentesting phase:

  1. Ping an IP address

    Ping can be used during a penetration test to send ICMP Echo Request packets to a target system or network, and then analyze the response to gather information about the target. By sending ping requests and analyzing the responses, pentesters can determine if a host is online, assess its response time (round-trip time or RTT), and identify potential targets for further exploration or exploitation.

  2. Fping

    fping is a program to send ICMP echo probes to network hosts, similar to ping, but much better performing when pinging multiple hosts

  3. Nmap Network Scan

    Nmap (“Network Mapper”) is an open source tool for network exploration and security auditing. It was designed to rapidly scan large networks, although it works fine against single hosts. Nmap uses raw IP packets in novel ways to determine what hosts are available on the network, what services (application name and version) those hosts are offering, what operating systems (and OS versions) they are running, what type of packet filters/firewalls are in use, and dozens of other characteristics. While Nmap is commonly used for security audits, many systems and network administrators find it useful for routine tasks such as network inventory, managing service upgrade schedules, and monitoring host or service uptime.

  4. Nmap Port Scan

    While Nmap has grown in functionality over the years, it began as an efficient port scanner, and that remains its core function. The simple command nmap target scans 1,000 TCP ports on the host target. While many port scanners have traditionally lumped all ports into the open or closed states, Nmap is much more granular. It divides ports into six states: open, closed, filtered, unfiltered, open|filtered, or closed|filtered.

  5. Traceroute

    Traceroute tracks the route packets taken from an IP network on their way to a given host.
